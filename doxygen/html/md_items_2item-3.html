Description

As a user I want to be able to generate 3 different occupancy percentages (25/50/100)
So that I can map out optimized seating plans

Acceptance criteria

### 1.

Given a basic plan (rows/columns) has been created
when the user chooses a percentage (25/50/100) from the menu
then the final plan (rows/columns/available seats) is generated.

### 2.

Given a basic plan has been created
when the user enters an invalid input
then the system askes for a valid input.

### 3.

Given the user has created the room
when the user wants to change the occupancy rate
then the user is denied and asked to create a new plan.
