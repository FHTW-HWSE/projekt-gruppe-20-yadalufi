 As a user~\newline
 I want to be able to define the room size by height and width (rows and cols)~\newline
 So that the plan matches the actual classroom\hypertarget{md_items_2item-1_autotoc_md5}{}\doxysection{\texorpdfstring{Acceptance criteria}{Acceptance criteria}}\label{md_items_2item-1_autotoc_md5}
\hypertarget{md_items_2item-1_autotoc_md6}{}\doxysection{\texorpdfstring{\#\#\# 1.}{\#\#\# 1.}}\label{md_items_2item-1_autotoc_md6}
Given that the user is on the room size configuration screen~\newline
 when the user types in the values of the room\textquotesingle{}s height and width in rows and columns as integers only~\newline
 then the system should accept the input and save the values in a struct.\hypertarget{md_items_2item-1_autotoc_md7}{}\doxysection{\texorpdfstring{\#\#\# 2.}{\#\#\# 2.}}\label{md_items_2item-1_autotoc_md7}
Given that the user has entered the values of the room\textquotesingle{}s height and width~\newline
 when the user hits enter,~\newline
 then the system should validate the input values to ensure that they are within an acceptable range and display an error message if the values are invalid.\hypertarget{md_items_2item-1_autotoc_md8}{}\doxysection{\texorpdfstring{\#\#\# 3.}{\#\#\# 3.}}\label{md_items_2item-1_autotoc_md8}
Given that the input values for the room\textquotesingle{}s height and width have been saved~\newline
 when the user selects \"{}show room\"{}~\newline
 then the system should display a graphical representation of the defined room size as a table that matches the actual classroom.\hypertarget{md_items_2item-1_autotoc_md9}{}\doxysection{\texorpdfstring{\#\#\# 4.}{\#\#\# 4.}}\label{md_items_2item-1_autotoc_md9}
Given that the user puts in values for the room\textquotesingle{}s height and width,~\newline
 when the input values are negative/invalid,~\newline
 then the system should refuse the values and display an error message.
